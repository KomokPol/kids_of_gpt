# Контракт для бота по Go-ядру

Этот файл задает правила для coding-agent при работе только с Go-частью проекта.

## 1. Роль бота

Бот реализует только Go-ядро и не вмешивается в Python/C++/frontend логику,
кроме чтения контрактов для интеграции.

## 2. Разрешенные пути изменения

- `gateway/`
- `services/go/`
- `proto/`
- `api/openapi/`
- `contracts/events/` (только если нужно для Go-фичи)
- `docs/` (при изменении поведения)

## 3. Запрещенные действия

- менять чужие стеки без явной задачи
- менять контракт и код в одном хаотичном PR
- добавлять скрытую бизнес-логику в gateway
- менять деньги вне wallet-ledger
- менять unlock-возможности вне progression-entitlements

## 4. Обязательные требования к каждому изменению

- объяснить зачем изменение
- зафиксировать контракт (если затронут API)
- добавить минимум один тест на критичную ветку
- не ломать совместимость без версии

## 5. Правила prompt для бота

Каждый prompt должен содержать:
- цель
- сервис
- список файлов, которые можно менять
- список файлов, которые нельзя менять
- ожидаемый результат

## 6. Критерий приемки

Изменение принимается, если:
- соблюдены инварианты денег и progression
- контракты и реализация совпадают
- есть понятный changelog в PR
